# 线上铺货 PC 交互 Demo Handoff

## Purpose

本目录把 APP 端「线上铺货对码工具」迁移为 PC 端视觉与交互 handoff。功能范围、可见文案和操作不扩展，只把 APP 的列表页、商品编辑区、候选集列表改成 PC 三栏并列。

## Source Inputs

- 产品指南：`线上铺货使用指南/线上铺货对码工具使用指南.docx`
- 版本补充：`线上铺货使用指南/生态店对码工具（线上铺货）---6月下（期待6.23上线）.docx`
- 最新专题：`线上铺货 app版需求文档和使用指南/生态店对码7月补充版本--100%准确率专题.docx`，覆盖前述文档中冲突的状态、标签、弹窗和同步规则。
- 旧版需求：`线上铺货使用指南/1.0.docx`、`线上铺货使用指南/2.0.docx`，仅用于补齐未被后续版本覆盖的前端交互。
- UI spec：未提供独立 PC 视觉规范，依据 Word 截图和全局布局规则推导。
- Real frontend project：未提供。本 handoff 不改生产项目。

## Output Mode

`visual-handoff`

当前工作区只有 Word 资料，没有真实前端工程、路由、组件库或 token 文件。因此本次生成独立 `index.html` 作为 PC 端交互参考，同时提供结构、状态、组件映射和实现说明，供后续转成真实前端。

## Structure Status

`structure-assumed`

用户已明确「APP 功能完全不变，只适配 PC 页面和交互」，资料也覆盖了主要流程；但尚未提供 PC 端已有设计系统、接口字段定义、真实前端目录和权限框架，相关内容在文档中标为实现假设。

## Discovered UI Constraints

- APP 端主色为高饱和蓝，核心按钮和选中态使用蓝色；PC 端只做密度和并列布局适配。
- 最新 APP 结构为「未匹配/待同步/已同步 tab + 搜索 + 筛选 + 商品卡片列表 + 详情页 + 候选集 + 弹窗」。
- 未匹配 `优先确认` 默认关闭；待同步的 `有标红/无标红` 互斥。
- 新版移除「待新建标品」列表状态；`status=1` 不应作为前端可见状态继续出现。
- 核对详情页要强化商品条码视觉层级，点击条码可打开补充美团 UPC 引导。
- 手动对码后的结果呈现统一：有成功候选集弹匹配成功；无成功候选集不再弹「匹配失败」，改成 UPC 已收录的安慰弹窗或打开聚焦引导页。
- APP 端大量信息在竖屏滚动中展开；PC 端只把列表、商品资料、候选集并列，不增加 APP 没有的说明文案或功能入口。
- PC 三栏宽度比例为 1/4、1/2、1/4。
- 用户改过任意商品资料后，在处方分类下出现「重置为初始商品资料」。
- 首次进入需要接受免责声明；页面提供 10 步教程。

## Open

直接用浏览器打开：

```bash
open pc-interaction-demo/index.html
```

如需静态服务：

```bash
python3 -m http.server 8080
```

然后访问 `http://localhost:8080/pc-interaction-demo/`。

## Included Screens And States

- 未匹配列表：`优先确认` 默认关闭；搜索、快捷筛选、商品卡片。
- 待同步列表：蓝黄对照、差异标红、库存未同步、单条同步、全部勾选和批量同步。
- 已同步列表：蓝黄对照、库存同步中、重新匹配和解除关联。
- 商品核对详情：商品编码、可编辑商品条码、品牌、通用名、商品名称、规格、批准文号、厂家、处方分类。
- 补充美团 UPC 引导：两张美团后台参考图、输入校验、合规后重新匹配、仅退出。
- 外卖候选集：高度相似、疑似重复、中等相似、略微相似、当前关联。
- 已匹配候选集：当前关联置顶；点击当前关联只确认关闭，点击其他候选先确认重新匹配。
- 未匹配候选确认弹窗：确认建联、发现为重复商品，主操作为匹配并同步。
- 手动重新匹配成功弹窗：有成功候选时提示匹配成功。
- UPC 已收录安慰弹窗：替代旧的匹配失败弹窗。
- 已匹配核查：我的商品/外卖商品对照、差异高亮、解除当前关联确认。
- 下一条：由前端记录进入列表时的后端返回顺序，但不在界面额外解释。
- 免责声明：首次 5 秒锁定、门店+账号版本记录、底部再次查看立即可关闭。
- 教程：10 步浮层，支持退出引导。

## Verification

已使用本机 Chrome headless 渲染并验证：

- `screenshots/desktop-1440x900.png`
- `screenshots/desktop-1280x800.png`
- `screenshots/desktop-1440x900-pending.png`

检查结果：

- 页面整体不产生横向或纵向滚动。
- 三栏在 1440x900 和 1280x800 下严格为 1:2:1。
- 抽样检查按钮、标签、标题、候选商品名无文本溢出。
- 可见文案中无 `O2On`、`商品序列`、`当前序列`、`自动对码标签`、`说明`、`查看详情`、`导出`、`对码工作台` 等非 APP 页面文案。
- 未匹配列表商品卡片不展示筛选或候选标签；待同步/已同步的同步状态只出现在外卖商品名称旁。
- 待同步/已同步蓝黄「我的商品 / 外卖商品」对照在左侧列表卡片内，详情页不展示。
- 右侧候选集卡片为图片 1/2、文字 1/2，图片区域明显变大。
- 点击商品条码输入框不会打开抽屉；只有点击「商品条码」四个字打开抽屉。
- 点击「点击这里查看详细指南」会打开同一个补充美团 UPC 抽屉。
- 修改任意商品资料后出现「重置为初始商品资料」，点击后恢复初始资料且按钮消失。
- UPC 不合规输入展示「请输入合规upc」。
- 合规 UPC 可触发匹配成功弹窗。
- 未匹配候选点击后弹「确认建联」或「发现为重复商品」，主操作文案为匹配并同步。
- 待同步/已同步当前关联候选弹出当前关联标品，非当前候选弹出重新关联确认。
- 待同步单条/批量同步会流转到已同步；解除关联会流转回未匹配。
- 首次免责声明 5 秒后解锁，再次打开立即可接受；10 步教程可完整走完。
- 1440x900 三栏为 348/696/348；1280x800 为 310/620/310；页面无整体溢出，控制台无错误。

## Migration Boundary

`index.html` 是视觉和交互参考，不是生产代码。真实实现时应复用项目已有的路由、表格/列表、表单、抽屉、弹窗、按钮、消息提示、权限和 API 请求层；本 demo 中的样例数据、CSS 类名和本地 JS 状态机仅用于说明交互。
